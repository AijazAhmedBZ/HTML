Summary - Chapter 4

  • To creat hyperlinks in HTML, we use anchor tags. < a> < /a>.
  • The text between the opening < a> tag and the closing < /a> tag is called the link text.
  • There are two basic choices when it comes to the selection of URLs.
  • Absolute URL - Usually used to navigate to another website.
  • Relative URL - Usually used to navigate within the same website.
  • Anchor elements always carry an attribute called href (Hypertext Reference). The value of
    the href attribute is the URL that you want people to go to when they click on the link text
    (The value of the href attribute becomes the link).
  • Either Relative or Absolut URLs may be used inside the href attribute.
  • When linking to a different website, the value of the href attribute will be the full
    web address for the website (including the domain name). This is the Absolute
    URL.
  • When you're linking to other pages within the same site, you can use Relative
    URLs. Relative URLs are like a short-hand version of Absolut URLs because
    there's NO NEED to specify the domain name.
  • Relative URLs help when building a site on your computer because you can creat
    links between pages without having to setup your domain name or hosting
  • To open a link in a new browser window or tab, you can use the target attribute in the
    opening < a> tag. The value of the target attribute should be "_blank".
  • The default value of the target attribute is "_self", which means the link will be
    opened in the current window or tab.

Summary - Chapter 5

  • The < img> element is used to add images to a webpage. This is an empty element (which means there is no closing tag).
  • The < img> element must carry the following two attributes:
    • src - Used to indicate the source of an image.
    • alt - Used to describe the content of an image.
  • The text used in the alt attribute is referred to as "Alt Text". The Alt Text should provide an accurate description
    of the image content so it may be understood accurately by screen reading software and search engines.
  • < img> is an inline element.
  • Images should be saved at the size you will be using them on the webpage and the format should be appropriate.
  • Photographs are the best saved as JPEG.
  • Illustrations or logos that use flat colors are better saved as GIF's.
  • To associate as < img> element with its caption, HTML5 has introduced a new < figure> element which
    can contain images and their captions so that the two are associated.
  • To add a caption to an image, the < figcaption> element may be used. This element is contained inside the < figure> element.

Summary - Chapter 7

  • Form controls live inside a < form> element. This element usually carries the following attributes:
    • Action attribute - The < form> element always carries the action attribute. It's value
      is the URL of the page that will receive the information when the form is submitted.
    • Method attribute - The form can be sent using one of two method: GET or POST.
      • GET - With the get method, the values from the form are added to the end
        of the URL specified in the action attribute. The following are further details
        about the get method:
        • The length of a URL is limited. (2,048 characters)
        • Never use get to sent sensitive data. (data will be visible in the URL)
        • GET method is better for non-secure data, like query strings in Google.
        • If the Method attribute is not used, the form data will be sent using the GET method.
        • Useful for form submissions where a user wants to bookmark the result.
      • Post - With the POST method,
        • Always use POST if the form data contains sensitive or personal
          information. (the POST method does not display the submitted data in
          the page address field)
        • POST has no size limitations and can be used to send large amounts of data.
        • Form submition with POST can not be bookmarked.
  • Data from a form is sent as name/value pairs.
  • Form controls may be used to collect data from users. Each form control is given name,
    and the text the user types in or the value of the options they select are sent to the server.
